# UI Components

## Shared Components
- navigation header
- search bar
- service cards
- booking timeline
- verification status badge
- review summary card
- notification inbox entry
- admin queue item

## Component Guidelines
- components should be reusable and context-agnostic where possible
- state and loading feedback should be explicit
- error states should explain the issue and next step
- preserve consistent spacing and interaction patterns

## Suggested Priorities
1. auth and onboarding components
2. search and profile cards
3. booking and messaging surfaces
4. admin review and moderation views
