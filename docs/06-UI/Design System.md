# UI Design System

## Purpose
The design system should provide a consistent, accessible, and modern experience across the Unamora web experience.

## Visual Principles
- clear hierarchy and simple navigation
- concise content with strong calls to action
- trust and clarity in every interaction
- support for both desktop and mobile contexts

## Foundations
- typography scale for headings, body text, metadata, and labels
- spacing system for layout rhythm
- color usage for trust, action, warning, and success states
- iconography and illustration rules

## Interaction Standards
- buttons should clearly communicate primary and secondary actions
- forms should provide inline validation and accessibility support
- dashboards should surface key status information quickly
- empty states should guide users toward next steps

## Accessibility Requirements
- WCAG 2.2 AA target
- keyboard support and visible focus states
- reduced motion support
- clear contrast and readable content
